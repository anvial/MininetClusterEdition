Mininet Cluster Edition

Expanding Mininet on Cluster Architecture

Download this project as a .zip file Download this project as a tar.gz file

Welcome to Mininet Cluster Edition Page.

Mininet Cluster Edition is a special software that expand Mininet prototypes for computer cluster, that allow us reproduce the network with such amount of nodes that hardly was possible before. The possible size of network topology in Mininet CE depends on number of cluster nodes with Mininet instances. One cluster node could simulate more than thousand hosts, and an modern server could execute at least 15 cluster nodes: we get about 15 thousands hosts by server. In collaboration that Mininet CE is well scaled system on we could simulated really big networks.

By the architecture, Mininet CE saves features of Mininet, so it do not become clear simulation system, it stay a net- work prototyping system. That means that we could trust the results of such simulation and there is no need to prove correctness and adequacy of built model.

Mininet CE Quick Start Guide

Creating Cluster Node VM

  • Create VM with two eth interfaces (1 - cluster data exchange, 1 - for control)

  • Install the Ubuntu 12.04 32-bit OS using ISO file from the ubuntu site link

  • Change the root password to "root":

        ~/$ sudo passwd root
  • Install the SSH:
        ~/# apt-get install ssh
  • Install python Scapy Module:
        ~/# apt-get install python-scapy
  • Install Mininet:
        ~/# apt-get install mininet
  • Clone this VM as much as you'd like nodes in you cluster.

Working With VM Clones

Some times you may face with problem of naming the eth interfaces in VM clone. To solve this problem you may do the following:

    ~/# rm /etc/udev/rules.d/70-persistent-net.rules
    ~/# reboot

Installing Mininet CE

  • Download the source code
        ~/$ git clone https://github.com/anvial/MininetClusterEdition.git
  • Install python NetworkX Module:
        ~/$ sudo apt-get install python-networkx
  • Install python Matplotlib Module:
        ~/$ sudo apt-get install python-matplotlib
  • Install python Paramiko Module:
        ~/$ sudo apt-get install python-paramiko

  • Download METIS lib METIS page.

  • Make METIS lib using README.TXT and INSTALL.TXT that included in distribution.

After that steps you can run Mininet CE using python2.7 interpreter

        ~/$python main.py

Simple Workflow

  • Fill the node list file (_config/nodelist.txt_) with IP addresses of VM in cluster. Example:
    10.20.30.1 root
    10.20.30.2 root
    10.20.30.3 root

  • Create (or use existing) input Mininet script with custom topology creation. The example of such script is in the standard Mininet distribution in "_custom/topo-2sw-2host.py_"

  • Run Mininet CE console

    ~/$ python main.py
  • Interact with Mininet cluster network. For example:
    mininet CE> help
    mininet CE> h1 ping h2
    mininet CE> h1 ifconfig
    mininet CE> hosts info
  • Close the Mininet CE
    mininet CE> exit
